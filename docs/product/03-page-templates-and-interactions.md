# 页面模板与交互原则

> 状态：阶段一 SSOT

## 1. 页面模板

### T1 今日工作台

结构：工作日期与状态摘要 → 紧急/异常队列 → 市场脉搏 → 运行任务 → 最近研究。

规则：首屏不超过 3 个同等视觉权重区域；异常按影响和可操作性排序；每个数字都能下钻到来源页。

### T2 列表与发现

适用：股票、题材、策略、任务、运行、研究记录。

结构：标题/状态 → 搜索筛选栏 → 结果摘要 → 高密度表格/列表 → 分页或虚拟滚动。

规则：筛选写入 URL；列排序明确；空态区分“没有数据”和“筛选无匹配”；整行可选但行内操作不冲突。

### T3 对象详情

适用：股票、题材、策略、任务、运行。

结构：对象头部 → 关键指标带 → 主结论区域 → tab/分区 → 证据与活动。

规则：对象身份、as-of、状态、更新时间和主操作在滚动前可见；tab 不重复对象头；详情接口可以并行加载，局部失败不拖垮整页。

### T4 分析工作区

适用：市场复盘、回测结果、运行比较。

结构：上下文控制 → 结论条 → 主图表 → 指标/分布/表格 → 诊断与口径。

规则：图表服务于问题；图例、单位、范围、缺失和基准明确；大表在容器内滚动并冻结表头。

### T5 配置与创建

适用：回测新建、声明式策略编辑、审核提案。

结构：对象选择 → 分组表单 → 实时校验 → 配置摘要 → 提交反馈。

规则：表单模型与 DTO 分离；后端 schema 为事实来源；提交期间禁止重复；失败保留输入；危险或不可复现配置需要显式警告。

### T6 运行与诊断

适用：数据运行、系统状态、任务详情。

结构：当前状态 → 时间线/步骤 → 影响范围 → 原因与证据 → 重试或下钻动作。

规则：stale/partial/failed 不使用模糊“异常”；显示最后成功时间、当前尝试、错误原文和用户可执行动作。

## 2. 通用页面状态

| 状态 | 视觉与行为 |
| --- | --- |
| initial loading | 保留页面骨架和上下文，不显示全屏 spinner；骨架尺寸接近真实内容 |
| refreshing | 保留旧数据，局部进度与“更新中”标识；禁止布局跳动 |
| empty | 说明是没有业务数据、没有权限还是筛选无结果，并提供下一步 |
| error | 保留可用上下文，展示稳定错误码/消息、重试和诊断入口 |
| stale | 数据仍可读，但明确最后成功时间、预期频率和风险 |
| partial | 明确缺失模块和影响，不将部分结果包装成完整成功 |
| success | 展示数据时间、版本/来源与必要警告，不用装饰性成功提示 |
| unauthorized | 区分未登录 401 与无权限 403；登录后恢复原 URL |
| not found | 区分对象不存在、被删除和不属于当前 owner，避免泄露他人资源 |

## 3. 时间与数据新鲜度

- 全局上下文栏显示：页面 as-of/交易日、最近数据时间、新鲜度、数据版本和警告数。
- “今天”仅用于当前自然日；历史市场数据使用明确交易日，如 `2026-07-16`。
- 日期选择默认取后端可用交易日，不假设自然日有行情。
- 每个 API 域通过 adapter 归一化 `asOf`、`updatedAt`、`source`、`warnings`；后端暂未提供时标记为 `unknown`，不得猜测。
- 图表 tooltip 和表格使用一致时区与格式；系统时间使用 ISO 8601，界面默认 Asia/Shanghai。

## 4. 主从对象与上下文

- 选中表格行时，优先通过 URL 或详情抽屉表达，不使用无法恢复的隐式选择。
- 从列表进入详情后返回，恢复筛选、排序、分页和滚动近似位置。
- 关联对象链接保留当前 as-of；例如从市场日进入股票详情携带 `date`。
- 比较集合、选中策略版本和运行 tab 必须可分享。

## 5. 表格

- 数字右对齐并使用 tabular numerals；名称/说明左对齐；状态居中或左对齐。
- 表头可排序时提供方向与键盘语义；默认排序写入 URL。
- 价格、百分比、金额、数量和日期使用共享格式化器；缺失统一显示 `—`。
- 红涨绿跌仅用于市场方向；错误使用错误红但搭配图标/文字，不能只靠颜色。
- 大表冻结表头；必要时冻结对象列；横向滚动局限在容器。
- 行内主动作不超过一个，其余放入更多菜单；禁止整页出现彩色图标墙。

## 6. 指标与卡片

- 指标带用于同一问题下的 4–8 个可比较指标，不把每个数字做成独立巨型卡片。
- 指标包含名称、值、单位、对比/变化、时间或口径入口；没有对比时不制造箭头。
- 卡片边界靠 1px 线和轻微底色，圆角 4–8px，阴影仅用于浮层和强调层级。
- 一屏内同等级卡片控制在少数，避免 Dashboard 瓦片化。

## 7. 图表

- 图表组件只接收展示模型；API DTO → domain model → series 转换放在纯函数。
- 不在图表组件内请求数据或计算后端权威绩效。
- 统一交叉线、tooltip、空态、颜色、坐标和 resize 生命周期。
- 净值/基准使用中性深绿/蓝；回撤使用低饱和红；红涨绿跌用于市场方向。
- 图表下方提供数据范围、采样/缺失、基准和计算版本。
- 支持键盘可达的 tab 和文本摘要；复杂图表至少提供关键指标表作为替代。

## 8. 表单与任务

- 表单字段按研究含义分组，不按 DTO 顺序罗列。
- schema 驱动字段仍需标签、描述、单位、范围、默认值来源和错误定位。
- 股票池模式切换会明确清理或保留哪些字段；不可见字段不得悄悄提交。
- 创建任务后立即进入任务详情；同步执行的后端也按任务状态展示，不伪造进度。
- 轮询采用退避和页面可见性控制；完成/失败后停止；刷新页面可恢复。
- 任务错误展示后端 detail，并提供“修改配置重试”，不自动重复创建任务。

## 9. Agent 交互

- Agent 从当前页面右侧抽屉打开；顶部固定显示作用域：对象、as-of、可用数据和工具权限。
- 提问默认带当前上下文，但发送前可查看/移除。
- 输出区分事实、推断、未知、反证和建议下一步；引用可打开对应证据。
- Agent 不能自动写入 canonical 事实、批准提案或更改回测结果。
- 没有后端 Agent 契约时显示能力说明，不使用本地假对话冒充正式功能。

## 10. 反馈与动效

- 交互过渡 120–180ms，只用于 hover、展开、抽屉和状态变化。
- 不使用大面积渐变、发光、漂浮、数字滚动或装饰性动画。
- Toast 仅用于瞬时操作结果；关键失败留在页面上下文中。
- 破坏性动作需要清晰对象名与后果；创建回测不属于破坏性动作，不滥用确认框。

## 11. 可访问性与键盘

- 所有控件有可见 focus；颜色对比达到 WCAG AA 的合理目标。
- 全局搜索支持 `/` 或 `Ctrl+K`，但快捷键不覆盖输入框默认行为。
- tab、菜单、抽屉和对话框具备正确语义、焦点管理和 Escape 行为。
- 图标必须有文字、tooltip 或 accessible name，不能靠图形猜测。
