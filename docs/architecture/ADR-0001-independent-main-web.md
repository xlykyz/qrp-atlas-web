# ADR 0001：新主 Web 独立建设

- 状态：Accepted
- 日期：2026-07-16

## 决策

在独立仓库 `qrp-atlas-web` 从零建设正式主 Web。旧 `qrp-atlas/web` 仅作为业务能力与 API 使用样本，不迁移其信息架构、布局、路由、状态管理和 UI 组件。

## 原因

旧前端按后端能力堆叠，存在 2,688 行个股页面、开发工具占据一级入口、业务状态与渲染耦合等问题。继续在其页面结构上改造会把临时产品模型固化为长期架构。

## 后果

- 三仓职责清晰，不建立跨仓库 UI 包。
- 旧 URL 仅在阶段四制定兼容与退出策略。
- 所有正式能力经 `qrp-atlas` API 使用。
- 新前端先验证黄金工作流，再扩展页面，避免批量生成低质量页面。
